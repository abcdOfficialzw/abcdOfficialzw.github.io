Kabukura.ai

Kabukura.ai serves as a specialized social media platform designed for Japanese investors seeking a more flexible and engaging alternative to Twitter. The platform emphasizes ease of use, with a clean, user-friendly interface that encourages interaction and connectivity among its users. Key features include a Twitter-like feed, options to like and dislike posts, insights into trending posts, user blocking functionality, and email notifications to keep users informed and engaged.

My Role

Position: Lead Frontend Developer
Responsibilities:
  • Implementation of the web design from Figma into a functional user interface, ensuring a seamless transition from design to web.
  • Development of responsive designs for both desktop and mobile views to cater to all users.
  • Integration of dynamic features like feeds, like and dislike functionalities, trending posts insights, and block user capabilities.
  • Establishment of email notifications to enhance user engagement and retention.

Technologies Used

  • Frontend: Developed using WeWeb, a powerful no-code platform that allowed for rapid development and deployment of the site's frontend, ensuring a high-quality, responsive design that adapts to both desktop and mobile views.
  • Backend: Build in XANO, a powerful nocode backend tool, providing a scalable and secure infrastructure capable of handling user data, authentication, and complex functionalities like feed algorithms and email notifications.

Challenges and Solutions

  • Dynamic Features Integration: Implementing features like the feed, like/dislike functionalities, and trending insights required thorough understanding and use of Database Architecture along with WeWeb's visual programming capabilities.

Project Impact

The platform was positively received by the target audience, providing Japanese investors with a tailored social media experience that caters specifically to their interests and needs. 12 hours after deployment the system reached an unexpected number of active users which led to the need to scale up our server infrastructure.
